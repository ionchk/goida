# Telegram-бот «Помощник студента: сдача ДЗ»

Простой бот для колледжа: помогает студенту сдать домашнее задание
преподавателю одним диалогом — выбрал предмет, написал комментарий,
прикрепил файл/фото, подтвердил отправку. Преподаватель сразу получает
уведомление.

## Возможности

- `/start` — приветствие и список команд
- `/submit` — пошаговый сценарий сдачи ДЗ (предмет → комментарий → файл → подтверждение)
- `/list` — мои последние 10 сдач
- `/help` — справка
- `/cancel` — отменить текущее действие
- Автоматическая пересылка домашки преподавателю в его чат с ботом
- Лог всех сдач в файл `submissions.json` (создаётся рядом со скриптом)

## Стек

- Python 3.10+
- [python-telegram-bot](https://github.com/python-telegram-bot/python-telegram-bot) ≥ 21
- Только стандартная библиотека для хранения (JSON)

## Установка

```bash
cd telegram-bot
python3 -m venv .venv
source .venv/bin/activate          # на Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

## Настройка

1. Создайте бота в Telegram через **@BotFather** и получите токен.
2. Узнайте `chat_id` преподавателя:
   - попросите его написать **@userinfobot** в Telegram, либо
   - временно запустите бота, попросите препода нажать `/start` и в логах
     посмотрите `update.effective_chat.id`.
3. Скопируйте файл с переменными окружения и заполните его:

   ```bash
   cp .env.example .env
   ```

   Откройте `.env` и впишите `BOT_TOKEN` и `TEACHER_CHAT_ID`.

## Запуск

Вариант 1 — через `.env` (нужен пакет `python-dotenv`):
```bash
pip install python-dotenv
python3 -c "from dotenv import load_dotenv; load_dotenv(); import bot; bot.main()"
```

Вариант 2 — экспорт переменных вручную (без лишних зависимостей):
```bash
export BOT_TOKEN="000000000:AAA..."
export TEACHER_CHAT_ID="123456789"
python3 bot.py
```

Бот работает в режиме long-polling: достаточно держать процесс запущенным
на любом сервере (или локально для проверки).

## Как пользоваться (сценарий студента)

1. Открыть бота в Telegram, нажать `/start`.
2. Отправить `/submit`.
3. Нажать на кнопку с нужным предметом.
4. Написать комментарий (например, «ЛР №5, вариант 1») или отправить `-`,
   если комментарий не нужен.
5. Прислать документ или фото домашки. Если вложения нет — отправить `/skip`.
6. Нажать **Отправить**.

Преподаватель получит сообщение со студентом, предметом, комментарием и
вложением.

## Структура

```
telegram-bot/
├── .env.example       — шаблон конфигурации
├── .gitignore
├── README.md          — этот файл
├── bot.py             — исходник бота
└── requirements.txt   — зависимости
```

При работе бот создаёт файл `submissions.json` рядом с собой — он добавлен
в `.gitignore` и в репозиторий не попадает.

## Безопасность

- Никогда **не** коммитьте `BOT_TOKEN` в репозиторий.
- Файл `.env` в `.gitignore` — это специально, не убирайте.
- Если случайно опубликовали токен — отзовите его в @BotFather через
  `/revoke` и сгенерируйте новый.
